# Release Checklist

Do not treat this checklist as proof by itself. It must point to real evidence.

- [ ] Core flow has been tested
- [ ] Error behavior has been tested
- [ ] Build or startup method is confirmed
- [ ] High-risk items are listed
- [ ] Secrets are not committed
- [ ] Rollback or recovery method is documented
- [ ] Nontechnical acceptance script is written
- [ ] Remaining untested items are classified
- [ ] Owner approval is recorded if needed
