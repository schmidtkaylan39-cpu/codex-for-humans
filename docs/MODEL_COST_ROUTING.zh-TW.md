# Model Cost Routing

這份規則的目的，是讓新手不要每件事都丟給最貴模型，也不要為了省錢把高風險判斷交給便宜模型。

## 一句話

能便宜做草稿，就便宜做草稿；需要本機證據，就交給 Codex；需要最後審查，才請 Web GPT Pro / Claude；碰到真錢、正式上線、secrets，就先停下來要人工批准。

## 它能做什麼

- 幫你判斷任務該用便宜模型、標準 Codex、高階審查，還是人工批准。
- 幫你記錄為什麼這樣分配。
- 幫你避免小任務浪費高階模型。
- 幫你避免高風險任務被便宜模型亂判。

## 它不能做什麼

- 不能保證真的自動切換模型。
- 不能自動使用你的付費帳號。
- 不能自動購買 token 或呼叫付費 API。
- 不能把外部模型的意見當成本機測試證據。
- 不能繞過人工批准。

## 四層分流

| 層級 | 適合用在 | 不適合用在 |
| --- | --- | --- |
| 便宜 / 本機 / mini | 摘要、草稿、文案、低風險檢查、重複資料整理 | 最終交付判斷、安全結論、交易、金流、正式上線 |
| 標準 Codex | 改檔、測試、除錯、讀 repo、整理證據、產出交付報告 | 需要第二審查的大型高風險決策 |
| 高階審查 | 架構審查、最終交付審查、高風險檢查、嚴重卡關 | 小文案、小修正、低風險雜事 |
| 人工批准 | 真錢、真交易、正式上線、secrets、付費工具、不可逆操作 | 可用假資料或本機模擬完成的普通任務 |

## 使用方式

日常使用時，不要重貼整份 SOP。直接貼：

```text
使用 $nontechnical-codex-project-controller
幫我先做 Model Cost Routing Check，再用最小安全成本執行這個任務。
```

70-100 交付檢查時，貼：

```text
使用 $nontechnical-project-readiness-auditor
幫我先做 Review Cost Routing，再檢查目前離交付還差什麼。
```

## 什麼時候一定要升級

- 有真錢、真交易、正式上線、真用戶資料。
- 有 secrets、API key、token、cookie、密碼、私鑰。
- 有金流、交易、登入、權限、安全設定、資料庫變更。
- Codex 已經連續修錯或判斷改變。
- 要做正式交付、公開發布、客戶驗收。

## 建議紀錄

可以在專案放一份：

```text
MODEL_ROUTING_LOG.md
```

用來記錄：

- 這次任務為什麼不用最貴模型
- 哪些部分能便宜處理
- 什麼情況要升級
- 是否需要人工批准

這份紀錄不能放 secrets。
